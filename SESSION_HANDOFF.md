# セッション引き継ぎ資料 — 株価テクニカル分析サイト

## プロジェクト概要

日本株テクニカル分析サイト。GitHub Pages でホスティング（静的HTML）。  
毎日18:00 JST に GitHub Actions で自動分析し、結果を `data/results.json` に保存。マッチ銘柄があればメール通知。

**リポジトリ:** `yagiyagisansam/keiba-ev`  
**開発ブランチ:** `claude/stock-technical-analysis-site-A6VMz`  
**本番URL:** `https://yagiyagisansam.github.io/keiba-ev/`

## 技術仕様（重要）

### フロントエンドの仕組み
- 純HTML/CSS/JS（フレームワークなし）
- チャート: TradingView Lightweight Charts v4.1.3 (CDN)
- データ: `data/results.json` を `fetch()` で読み込み（キャッシュバスター付き）
- 銘柄管理: `localStorage`（フロントエンド側）/ `data/stocks.json`（分析側）
- メソッド選択: `localStorage` に `enabledMethods` セットを保存
- レスポンシブ: 600px未満→ボトムシート、600px以上→左オーバーレイ

### analyze.py の仕組み
- `yfinance` でデータ取得（`{code}.T` 形式、period="1y"）
- 指標計算: MA5/25/75、出来高MA25、RSI14、MACD、Bollinger Bands
- 全手法を0/1の二値で判定（スコアリングなし）
- チャートデータ: 最新90日分を `results.json` に保存
- メール: マッチ銘柄がある場合のみ送信（Gmail SMTP、環境変数から認証情報取得）

### results.json フォーマット

### UI設計の確定事項
- **「確度」は一切表示しない**（ユーザーのオフライン判断用のみ）
- チェックボックス = 分析に使う手法の選択（マッチを示すものではない）
- マッチした銘柄を上部に表示 → マッチした手法のバッジ → 説明図 + 日足チャート
- 出来高（C16/17/18）はバッジの下に「出来高急増（前日比180%）」のような注釈テキストのみ

## 会話の文脈で重要なポイント

- 「確度」という言葉はユーザーが手法選別に使う個人的判断基準。サイト・コード・通知に一切表示しない
- 出来高手法はスタンドアロンではなく、他の手法がマッチしたときの「補強注釈」として機能
- yfinance は無料で安定、Yahoo Finance 経由（`{code}.T`）
- 最大銘柄数100、最大手法数30
- スマホ対応必須（ボトムシート形式のドロワー）
